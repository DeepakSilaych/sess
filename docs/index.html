sess/

dtach sessions with overlay isolation.
One tool. No worktrees. No containers. No tmux.

$ npx @deepaksilaych/sess

or npm install -g @deepaksilaych/sess  ·  source

dtach persistence

Single Unix socket per session. Ctrl+\ to detach. Sessions survive SSH drops and laptop sleep.

OverlayFS isolation

N independent writable views of the same repo. Only deltas stored. Linux kernel-native.

APFS clones on macOS

Same command, different kernel primitive. Copy-on-write clones — instant, isolated.

Connection log

Every session end classified: detach, exit, or drop. Know what happened while you were away.

Remote sessions

sess remote add once. Then sess new, sess <name>, sess ssh all go through SSH automatically.

Auto-reconnect on wake

sess up reconnects to all previously active sessions. SleepWatcher triggers it on macOS wake.

Commands

sess new feature-auth # create + auto-attach sess feature-auth # reattach (Ctrl+\ to detach) sess ls # list sessions sess rm feature-auth # destroy session + overlays sess overlay new feature-auth bugfix main sess overlay switch feature-auth bugfix sess overlay ls feature-auth sess overlay rm feature-auth bugfix sess diff feature-auth # git diff in overlay sess log feature-auth # activity log sess connections feature-auth # detach/exit/drop log sess path feature-auth # print overlay path sess code feature-auth # open Cursor / VS Code sess ssh # SSH to configured VM sess up # reconnect all active sessions sess remote add default user@dev-vm sess doctor # check prerequisites
