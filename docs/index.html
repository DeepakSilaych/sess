What is sess?

sess gives you persistent terminal sessions with filesystem isolation. Create a session, work inside it, detach with Ctrl+\. The session survives SSH drops, laptop sleep, and network failures. Reattach anytime with sess <name>.

On Linux, each session gets an OverlayFS mount — an independent writable view of your repo. Changes in one session never leak to another. On macOS, APFS copy-on-write clones provide the same isolation.

Quick start

sess new feature-auth          # create session + auto-attach
# ... work inside the session ...
Ctrl+\                            # detach (session persists)
sess feature-auth                 # reattach

sess overlay new feature-auth bugfix main
sess overlay switch feature-auth bugfix
# ... work on bugfix in isolated overlay ...
Ctrl+\

sess rm feature-auth              # destroy session + overlays

Commands

Every command in sess. All commands work locally and over SSH when a remote is configured.

Sessions

sess new <name> [branch]        # create session + auto-attach
sess <name>                     # attach to existing session
                                # Ctrl+\ to detach
sess ls                          # list all sessions
sess rm <name>                    # destroy session (unmounts overlays)
sess status [name]                # session info or overall status
sess path <name>                   # print overlay/cwd path (for scripts)

Overlays

sess overlay new <session> <name> [branch]
sess overlay switch <session> <name>

Inspection

sess log <name> [N]               # activity log (default: last 20)
sess connections <name> [N]        # connection log (detach/exit/drop)

Remote

Overlays

Overlays give each session an independent writable view of your repository. Changes in one overlay never affect the original repo or other overlays.

How it works

On Linux, sess uses OverlayFS — a union filesystem built into the kernel. It stacks a writable layer on top of a read-only base:

  • Lower layer — the original repo (read-only, never modified)
  • Upper layer — your changes (only deltas are stored)
  • Merged view — what you see and work in (lower + upper combined)

On macOS, sess uses APFS copy-on-write clones (cp -cR). Same experience — isolated writes, shared disk blocks for unchanged files.

Creating an overlay

sess overlay new feature-auth bugfix-1222 main

This:

  1. Pulls the latest main branch from origin
  2. Creates an OverlayFS mount (Linux) or APFS clone (macOS)
  3. Checks out the specified branch inside the overlay
  4. Updates the session state to point to the new overlay

Switching overlays

sess overlay switch feature-auth bugfix-1222

Switches the active overlay for the session. Detach and reattach to update your working directory.

Listing overlays

sess overlay ls feature-auth
bugfix-1222          main          mounted    <- current
feature-x            main          mounted

Removing an overlay

sess overlay rm feature-auth bugfix-1222

Unmounts the overlay (Linux) and removes the overlay directory. If this was the current overlay, the session resets to none.

Isolation in practice

# Create overlay and write to it
sess overlay new my-session ov1 main
echo "isolated change" >> ~/.sess/sessions/my-session/overlays/ov1/merged/README.md

# Original repo is untouched
cat /path/to/repo/README.md
# → "hello"  (no "isolated change")

# Diff shows overlay changes
sess diff my-session
# → +isolated change

Directory structure

~/.sess/sessions/<name>/overlays/<overlay>/
├── upper/      # writable layer (deltas only)
├── work/       # kernel work directory
└── merged/     # unified mount point (what you see)

Privilege requirements (Linux)

OverlayFS mounts require CAP_SYS_ADMIN. Options:

  • sudo — sess uses sudo mount automatically
  • setcap — sudo setcap cap_sys_admin+ep $(which sess)
  • User namespaces — if enabled on your kernel

Remote sessions

Sessions live on a Linux VM. Your laptop is just a terminal. sess handles all SSH automatically.

One-time setup

sess remote add default user@dev-vm

This stores the SSH host in ~/.sess/remote. Key-based SSH auth is required — set it up first with ssh-copy-id user@dev-vm.

Daily workflow

# Create a session (happens on the VM)
sess new feature-auth

# Reattach (SSHes to VM automatically)
sess feature-auth

# Just SSH to the VM (no session)
sess ssh

# Open Cursor/VS Code via Remote-SSH
sess code feature-auth

Auto-reconnect on wake

When you close your laptop, SSH drops but dtach sessions survive on the VM. When you open it:

sess up

This reconnects to all previously active sessions. On macOS, it opens a Terminal.app tab for each session.

Fully automatic (macOS)

Install SleepWatcher and add to ~/.wakeup:

#!/bin/bash
# ~/.wakeup — called by SleepWatcher on wake
sleep 3 && sess up &
brew install sleepwatcher
cp ~/Projects/sess/etc/wakeup ~/.wakeup
chmod +x ~/.wakeup

Connection log

Every time a session ends, sess records how it ended. This tells you whether you intentionally detached or your connection dropped.

Viewing the log

sess connections feature-auth
2026-07-09T14:30:00+05:30  detach (Ctrl+\)
2026-07-09T15:12:00+05:30  drop (exit code 141)
2026-07-09T15:12:30+05:30  detach (Ctrl+\)
2026-07-09T16:45:00+05:30  exit

Event types

EventMeaningHow it happens
detachYou intentionally detachedPressed Ctrl+\
exitYou intentionally exitedTyped exit or Ctrl-D in the shell
dropUnintentional disconnectSSH timeout, network failure, laptop sleep

How detection works

sess checks dtach's exit code after the session ends:

  • Exit code 0 — the shell exited cleanly → exit
  • Exit code 1 — dtach received the detach signal → detach
  • Anything else — abnormal termination → drop

Log location

~/.sess/sessions/<name>/connections

One entry per session end. Timestamped in ISO 8601. Plain text, append-only.

Activity log

Separate from the connection log, the activity log records all session events:

sess log feature-auth
2026-07-09T11:19:57+00:00  created (branch: master)
2026-07-09T11:19:57+00:00  attached
2026-07-09T11:19:57+00:00  detached (Ctrl+\)
2026-07-09T11:21:00+00:00  attached
2026-07-09T11:22:00+00:00  overlay new: bugfix-1222 (branch: main)
2026-07-09T11:23:00+00:00  overlay switch: bugfix-1222
2026-07-09T11:25:00+00:00  diff
2026-07-09T11:30:00+00:00  connection dropped (exit code 141)

Stored at ~/.sess/sessions/<name>/log.
