What it does

Six things, done right.

▢

dtach persistence

Single Unix socket per session. No server process. Ctrl+\ to detach, sess <name> to reattach. Sessions survive SSH drops, laptop sleep, network failures.

◈

OverlayFS isolation

N independent writable views of the same repo. Only deltas stored. Writing to one overlay never affects the original or other overlays. Linux kernel-native, instant creation.

⬡

APFS clones on macOS

Same sess overlay new command, different kernel primitive. Copy-on-write clones on APFS — instant, isolated, shared disk blocks for unchanged files.

≣

Connection log

Every session end is classified: detach (you pressed Ctrl+\), exit (you typed exit), or drop (SSH timeout, network failure). Know what happened while you were away.

⇄

Remote sessions

sess remote add once. Then sess new, sess <name>, and sess ssh all go through SSH automatically. Sessions live on the VM. Your laptop is just a terminal.

⚡

Auto-reconnect on wake

sess up reconnects to all previously active sessions. On macOS, SleepWatcher triggers it automatically when you open your laptop. Zero commands to type.

Reference

Every command.

$ sess new feature-auth # create session + auto-attach $ sess feature-auth # attach to existing session # Ctrl+\ to detach $ sess ls # list sessions $ sess rm feature-auth # destroy session + overlays $ sess overlay new feature-auth bugfix main # create overlay from latest main $ sess overlay switch feature-auth bugfix # switch active overlay $ sess overlay ls feature-auth # list overlays $ sess overlay rm feature-auth bugfix # remove overlay $ sess diff feature-auth # git diff in overlay $ sess log feature-auth # activity log $ sess connections feature-auth # connection log $ sess path feature-auth # print overlay path (for scripts) $ sess code feature-auth # open Cursor / VS Code $ sess status # overall status $ sess ssh # SSH to configured VM $ sess up # reconnect all active sessions $ sess remote add default user@dev-vm # configure remote VM $ sess doctor # check prerequisites $ sess help # show help

Platform

Works where you work.

🐧 Linux

Full OverlayFS support. Instant writable overlays. Delta-only disk usage. Requires root or CAP_SYS_ADMIN for mounts.

🍎 macOS

APFS copy-on-write clones as overlay fallback. dtach sessions, status bar, remote SSH, auto-reconnect via SleepWatcher.

🔌 Remote (SSH)

Sessions live on the VM. sess ssh connects. sess up reconnects on wake. Laptop is just a terminal.

⌨️ Editor

sess code <name> opens Cursor or VS Code. Remote sessions use Remote-SSH. On demand, never automatic.

Install

One command.

$ npx @deepaksilaych/sess copy
$ npm install -g @deepaksilaych/sess copy
$ git clone https://github.com/DeepakSilaych/sess.git && cd sess && sudo make install copy

Requires: dtach, git, bash. OverlayFS needs Linux. APFS clones need macOS.
